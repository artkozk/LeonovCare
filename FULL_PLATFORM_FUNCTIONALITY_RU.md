# LeonovCare: полный функционал платформы (сайт + бот + сервисы)

Дата актуализации: 13.04.2026  
Источник: текущий код и документация репозитория `LeonovCareSite` (без изменений кода бота).

---

## 1) Что входит в платформу

### 1.1 Сайт (маркетинговый контур)
1. Статический сайт на чистом `HTML/CSS/JS` без SPA-фреймворка.  
   Детали: быстрый деплой, предсказуемый рендер, SEO-упор, минимум операционных рисков.

2. Контент и бизнес-логика тарифов централизованы в `config.js`.  
   Детали: офферы, цены, тарифные карточки, направления, зарплатный график, API-отзывы, ссылки на ментор/канал/бот.

3. Единый runtime в `script.js` обслуживает все страницы.  
   Детали: рендер секций, кнопки, deep-link в бот, калькулятор автооткликов, отзывы, FAQ, мобильное меню, smart-header, reveal-анимации.

4. Страницы услуг и направлений.  
   Детали: `index.html`, `zero-offer.html`, `interview-prep.html`, `grade-salary.html`, `autoapply.html`, `directions.html`, `interview-helper.html`, треки `java/golang/frontend/python/php/qa/ml/analytics/devops/mobile/data-engineering`.

5. Deep-link маршрутизация в Telegram-бот из любой кнопки сайта.  
   Детали: payload формата `lc2_<route>_<service>_<plan>_<source>_c<clicks>`, где route: `home/student/enroll/materials/auto`.

6. Корзина тарифа в браузере (`localStorage`).  
   Детали: ключ `lc_tariff_cart_v1`, хранит выбранную услугу, план, цену, источник кнопки, отклики, рассрочку.

7. Калькулятор автооткликов на сайте.  
   Детали: `<200` = `7 ₽` (первые 50 бесплатно 1 раз), `200-499` = `6 ₽`, `>=500` = `5 ₽`.

8. Отзывы грузятся с внешнего API.  
   Детали: `https://api.it-mentors.ru/v1`, адаптация разных форматов ответа, fallback-состояния (загрузка/пусто/ошибка).

### 1.2 Telegram-бот (операционный контур)
1. Бот на `aiogram` (через внутренний адаптер `telebot`) + `SQLite` + scheduler.  
2. Polling-архитектура с авто-повторами при сетевых сбоях/конфликтах.  
3. Ролевая модель: `обычный пользователь`, `ученик`, `админ (full/limited)`, `owner-only /t`.  
4. FSM-состояния пользователей хранятся в БД (`user_states`).  
5. Все ключевые действия логируются в `event_log`, профили активности в `user_profiles`.

### 1.3 Смежные сервисы
1. `hh-autoresponder` — сервис автооткликов HH (internal API).  
2. `interview-assistant` — API для выпуска/проверки лицензий Interview Helper.  
3. `AutoHh/parser_hh` — парсинг резюме HH (поиск, фильтры, выгрузки).  
4. Cloudflare OpenAI Proxy (опционально) — проксирование LLM-запросов для `/t -> Отпис`.  
5. `systemd`-деплой бота (`mentor-bot.service`) + `.env`-конфиг.

## 2) Функционал для обычного пользователя (не ученик)

### 2.1 Вход и навигация
1. Стартовое меню с ключевыми продуктами.  
   Детали: `Вступить на обучение`, `Бесплатные материалы`, `Автоотклики`, `Interview helper`, `Лицензия Jetbrains`, `Диагностический созвон`, `Я ученик`.

2. Приход с сайта сразу в нужный раздел через deep-link.  
   Детали: бот парсит payload `lc2_*`, открывает нужный route и пишет аналитику `app.start_deeplink`.

3. Автосохранение `site intent` (намерения лида).  
   Детали: сохраняется route/service/plan/source/clicks для аналитики маркетинга и продаж.

### 2.2 Вступление на обучение (enrollment)
1. Выбор уровня и трека.  
   Детали: `С нуля` или `Есть база`; треки `С нуля до оффера` / `После курсов` / `Увеличение зарплаты`.

2. Выбор тарифа и фиксация условий.  
   Детали: бот показывает тариф, цену, условия, ведет к созданию договора.

3. Создание договора.  
   Детали: режимы:
   - статическая ссылка/файл договора;
   - HTTP-интеграция OkiDoki;
   - official OkiDoki endpoint с `api_key`, `template_id/entities`.

4. One-time политика договора.  
   Детали: автосоздание договора один раз на пользователя (`enrollment_contracts`), повтор через запрос ментору.

5. Оплата после договора через Cardlink.  
   Детали: создание checkout, кнопка `Оплатить`, проверка статуса `Проверить оплату`, автоматическое применение результата.

### 2.3 Автоотклики HH (autoapply)
1. Полный wizard запуска.  
   Детали: направление -> цель откликов -> запрос -> логин HH -> пароль -> подтверждение.

2. Core-направления с автозапросами.  
   Детали: `java/frontend/golang/python` могут идти без ручного текста (берется из админ-настроек).

3. Тарифная логика и бесплатные 50 откликов (1 раз).  
   Детали: хранится и контролируется через настройки/состояние пользователя.

4. Запуск через internal API автооткликов.  
   Детали: create account -> run account; поддержка обоих run endpoint (новый и legacy), recovery-логика.

5. Оплата автооткликов с сайта через Cardlink (если требуется).  
   Детали: проверка статуса в боте, после успеха — запуск автооткликов и фиксация операции.

6. Просмотр последнего запуска и краткой статистики.  
   Детали: статус, итоги, ошибки, снэпшоты create/run.

### 2.4 Interview Helper
1. Отдельная витрина внутри бота.  
   Детали: показ текущей лицензии, срока, кнопка оплаты/получения ключа, ссылка на скачивание приложения.

2. Платный сценарий для обычного пользователя.  
   Детали: `INTERVIEW_HELPER_PRICE_RUB` и `INTERVIEW_HELPER_DURATION_DAYS`, checkout через Cardlink.

3. Безопасная выдача ключа.  
   Детали: после оплаты бот выпускает ключ через API и отдельно валидирует статус лицензии.

### 2.5 Диагностический созвон
1. Запись в свободные слоты.  
   Детали: только будущие слоты, окно `09:00–19:30`, шаг `30 минут`.

2. Лимит для не-ученика.  
   Детали: 1 диагностический созвон на пользователя.

3. Управление записью.  
   Детали: перенос не позже чем за 24 часа, отмена не позже чем за 12 часов.

4. Обязательная загрузка PDF-резюме до брони.  
   Детали: без резюме слот не подтверждается.

5. Уведомления ментору.  
   Детали: при брони/переносе/отмене бот отправляет менторские уведомления.

### 2.6 Лицензия Jetbrains
1. Отдельный раздел `Лицензия Jetbrains`.  
   Детали: информационный/маршрутизирующий entrypoint из меню.

## 3) Функционал для ученика

### 3.1 Онбординг в роль ученика
1. Вход по одноразовому коду (`Я ученик`).  
   Детали: коды с TTL или без срока, статусы `active/used/expired/revoked`.

2. После валидного кода открывается кабинет ученика.  
   Детали: роль пользователя переводится в `student`, строится отдельный маршрут интерфейса.

### 3.2 Кабинет ученика
1. `Моя анкета`  
2. `Доступы HH`  
3. `Бесплатные материалы`  
4. `Созвоны с ментором`  
5. `Моки`  
6. `Автоотклики`  
7. `Interview helper`  
8. `Лицензия Jetbrains`  
9. `Подтвердить платёж` (только при корректном контексте постоплаты)

### 3.3 Анкета ученика (profile wizard)
1. Пошаговый сбор данных.  
   Детали: ФИО, username, направление, этап, тариф, предоплата, постоплата, комментарий, дата входа, ссылка на договор.

2. Ручной режим и prefill из договора.  
   Детали: бот умеет подтянуть данные по ссылке OkiDoki/contract payload/PDF-hints.

3. Отправка анкеты ментору на модерацию (`PROFILE`).  
   Детали: заявка уходит всем админам, дальше approve/reject.

4. После одобрения данные превращаются в карточку ученика.  
   Детали: создается/обновляется запись в `students`, привязка owner к TG ID.

### 3.4 Доступы HH
1. Ученик отправляет HH-реквизиты (`ACCOUNTS`) на подтверждение.  
2. Передаются поля: телефон, email, пароль, дата рождения, локация, зарплатные ожидания min/comfort.  
3. После одобрения данные записываются в карточку ученика.

### 3.5 Платежи ученика
1. Подтверждение платежа учеником (`PAYMENT`).  
   Детали: теперь сценарий ориентирован на Cardlink-checkout, ручные чеки не обязательны при активной кассе.

2. После подтверждения вносится postpay-платеж в БД.  
   Детали: сдвигается `next_due_date`, обновляется `student_next_notify_date`, долг может авто-закрыться.

### 3.6 Материалы и roadmap
1. Доступ к бесплатным материалам с подписочным гейтом.  
   Детали: проверка подписки через `get_chat_member`, fallback на invite URL.

2. Прогресс по шагам.  
   Детали: отмечать `done/undone`, статистика по направлению/блоку/общая.

3. Витрина всех направлений.  
   Детали: отдельный экран с большим каталогом (включая дополнительные 22+ трека/базу/карьерные блоки).

### 3.7 Созвоны с ментором
1. Бронирование/перенос/отмена собственных CALL-событий.  
2. Ограничения по времени и окнам как в правилах системы (`24ч` перенос, `12ч` отмена).  
3. В карточке созвона появляется кнопка подключения, когда ссылка назначена.

### 3.8 Моки (парный формат)
1. Поиск напарника по направлению.  
   Детали: `waiting -> paired -> scheduled`.

2. Выбор слота и установка ссылки на мок.  
3. Подписка на уведомления о моках для учеников.  
4. Рассылка уведомлений участникам и подписчикам, когда мок готов.

### 3.9 Автоотклики ученика
1. Ученик может запускать автоотклики из своего кабинета.  
2. Если логин уже известен — упрощенный вход.  
3. Если запуск требует оплаты — встроенный checkout flow.

### 3.10 Interview Helper для ученика
1. Если включен `INTERVIEW_HELPER_STUDENT_FREE`, ключ выдается/продлевается бесплатно.  
2. Иначе работает платный путь через Cardlink, как у обычного пользователя.

## 4) Функционал для ментора / админа

### 4.1 Роли админов и права
1. Базовые админы из конфига (`ADMIN_IDS`) + динамические админы из БД.  
2. Грейды: `full` и `limited`.  
3. Тонкие permission-флаги: управление материалами, рассылками, админами, платежной статистикой.

### 4.2 Главная панель ментора
1. `Ученики`  
2. `Напоминалки`  
3. `Уведомления`  
4. `Платежи`  
5. `🔑 Доступы HH`  
6. `🎟 Коды ученика`  
7. `📚 Материалы` (по правам)  
8. `📣 Сообщения` (по правам)  
9. `👮 Админы` (по правам)  
10. `Статистика`

### 4.3 Ученики (CRM-контур)
1. Список + карточки + карусель.  
2. Поиск по ФИО/username/HH/комментарию.  
3. Фильтры: направление, этап, тариф, оффер, архив, сортировки (включая дату вступления/последнего пополнения).  
4. Добавление ученика wizard-ом с защитой от дублей ФИО/username.  
5. Карточка хранит: тариф, оффер, pre/post суммы, график постоплаты, договор, HH-данные, источники лида, собесы и пр.  
6. Редактирование всех полей карточки, включая приватные HH поля и ссылку договора.  
7. Архив/восстановление + фиксация причины провала сделки.  
8. Постоплата: настройка `% всего`, ежемесячных правил, уведомлений ученику, enable/disable.

### 4.4 Коды ученика
1. Генерация одноразовых кодов на 24 часа или без срока.  
2. Просмотр статусов по группам.  
3. Ручной отзыв активных кодов.

### 4.5 Доступы HH + автозапросы
1. Реестр учеников с заполненными HH данными.  
2. Просмотр пароля с режимом hide/show.  
3. Настройка autoapply query для core направлений (`java/frontend/golang/python`) + reset to default.

### 4.6 Напоминалки
1. Персональные и привязанные к ученику.  
2. Частоты: `daily`, `every_n_days`, `weekly`, `once`.  
3. Редактирование title/note/time/frequency/days.  
4. Enable/disable и удаление.

### 4.7 Правила напоминаний по этапам
1. `stage_reminder_rules` для каждого этапа обучения.  
2. Массовое применение правила ко всем активным ученикам этапа.  
3. Автосинхронизация `student_stage_reminders`.

### 4.8 События/уведомления
1. Типы событий: `HR/TECH/CALL/OTHER`.  
2. Календарные представления: сегодня, неделя, все предстоящие, сетка слотов.  
3. Создание события через wizard с проверкой пересечений и force-overlap режимом.  
4. Редактирование любого поля события, удаление, смена ученика/типа/времени/длительности.  
5. Личные toggle-настройки admin-уведомлений: пинги, напоминалки, события, daily schedule.

### 4.9 Платежи
1. Витрины: `Ждём сегодня`, `На неделе`, `Долги`.  
2. Бюджет: месяц, последние 30 дней, произвольный период, по ученику, период по ученику.  
3. Расчет остатков постоплаты, рекомендации суммы платежа.  
4. Внесение prepay/postpay вручную с корректным обновлением графика postpay.  
5. Автозакрытие postpay при полном погашении.  
6. Встроенная диагностика интеграции Cardlink (`py:cardlink_check`).

### 4.10 Алерты и антиспам-поведение
1. Типы алертов: `REMINDER`, `EVENT`, `PAYMENT`.  
2. Действия на алерте: `Увидел`, `Snooze`, `Shift +1/+3`, перенос на дату, открыть ученика/событие, внести платеж.  
3. Для ученика `PAYMENT seen/shift` меняет напоминание, но не ломает сроки менторского учета.  
4. Scheduler чистит устаревшие pending-алерты и не зацикливает одноразовые reminders.

### 4.11 Модерация заявок (requests)
1. Обрабатываются типы: `PROFILE`, `ACCOUNTS`, `PAYMENT`, `SITE_INTENT`, `AUTOAPPLY`, `AUTOAPPLY_PAYMENT`, `ENROLLMENT`, `ENROLLMENT_PAYMENT`, `INTERVIEW_HELPER_PAYMENT`.  
2. По `PROFILE` обязательно выбирается источник лида (канал/холодные/сарафанка/ом/кастом).  
3. Одобрение заявки запускает прикладные действия: создание/апдейт карточки, платежи, выпуск лицензии, статусы договора и т.д.  
4. Отклонение отправляет пользователю причину и фиксирует события воронки.

### 4.12 Материалы (админ-контур)
1. Настройка подписочного гейта: `group_chat_id`, invite URL, require_subscription on/off.  
2. Редактор roadmap: add/insert/replace/edit/delete шагов.  
3. Preview по направлению.  
4. Sync из группы по topic/thread.  
5. Auto-detect thread и bulk import истории сообщений.

### 4.13 Сообщения и рассылки
1. Ручные рассылки по сегментам: `all/student/regular/admin`.  
2. Поддержка копирования форматирования и медиа через template payload/copy_message.  
3. История кампаний + sent/failed.  
4. Удаление уже отправленных сообщений у получателей.  
5. Автосообщения по неактивности: правило, порог дней, enable/disable, запуск сейчас.  
6. Опциональная кнопка `✅ Прочитано` и сбор read-метрик.

### 4.14 Статистика и аналитика
1. Разделы: `Обзор`, `Пользователи`, `Материалы`, `Я ученик`, `Платежи`, `Операционка`, `Неактивные ученики`.  
2. Метрики: DAU/WAU/MAU, stickiness, D1/D7/D30 retention.  
3. Воронки: материалы и student-entry.  
4. Платежная аналитика и топы учеников.  
5. Операционная аналитика алертов/ошибок/доставки.  
6. Экспорты: TXT/HTML отчеты + `event_log` выгрузки.

### 4.15 Управление админами
1. Просмотр базовых и динамических админов.  
2. Добавление full/limited по ID или username (если пользователь уже известен системе).  
3. Удаление одного/всех динамических админов (базовых удалить нельзя).  
4. Персональные настройки уведомлений на админа.

### 4.16 Interview Helper (админ-инструмент)
1. Админ-команда `/ih_reset_keys_all` деактивирует активные ключи.  
2. Используется для аварийного сброса лицензий.

## 5) Owner-only блок `/t` (владелец, суперюзер)

> Это отдельный сверх-уровень, не обычный админ. Доступ ограничен `OWNER_SUPERUSER_ID`.

### 5.1 Разделы `/t`
1. `🛠 Модерирование`  
2. `📣 Отпис`  
3. `⭐ Реакции`  
4. `📊 Общая статистика`  
5. `ℹ️ Как пользоваться /t`

### 5.2 Модерирование аккаунтов
1. Управление TG-аккаунтами для outreach.  
2. Типы аккаунтов: `standard` и `polygon`.  
3. Назначение основного аккаунта.  
4. QR-авторизация с поддержкой облачного пароля Telegram (2FA).  
5. Очистка неавторизованных аккаунтов и session-артефактов.

### 5.3 Отпис / outreach pipeline
1. Быстрый запуск и ручной запуск кампаний.  
2. Настройки: режим `все аккаунты`/`выбранные`, интервал, лимит на аккаунт.  
3. HH-фильтр, файлы контактов CSV/TXT, отчет кампании.  
4. Диагностика готовности (`парсер`, `cookies`, `TG auth`, `AI`, `active jobs`).  
5. Сброс зависшей активной задачи.

### 5.4 Скрипт касаний и AI-ответы
1. Редактор `Скрипт 4` по направлениям (`java/frontend/golang/python`) и шагам.  
2. Тумблер AI-автоответов + автообработка входящих ответов лидов.  
3. Интервал AI-цикла и strict mode.  
4. Guardrails: forbidden substrings/regex, max message chars.  
5. AI-тесты сценариев с генерацией TXT/HTML/JSON отчетов.

### 5.5 AI-инфраструктура
1. Режимы LLM: `openai` или `proxy`.  
2. Поддержка `OWNER_AI_BASE_URL`, `OWNER_AI_PROXY_AUTH`, `OWNER_AI_HTTP_PROXY`.  
3. Health-check proxy с кэшем статуса.  
4. strict mode: при проблемах LLM автоответы не уходят fallback-ом.

### 5.6 Антидубли и качество рассылки
1. Дедуп по Telegram контакту (`owner_outreach_contacted`).  
2. Пул спарсенных контактов (`owner_outreach_parsed_pool`) и добор только недостающего объема.  
3. Защита от параллельных запусков и конфликтов Telethon session.  
4. Учет статусов лида и подробные отчеты по аккаунтам/ошибкам/LLM.

### 5.7 Реакции на канал
1. Ручной запуск реакций (пресеты 5/20 постов).  
2. Авто-режим реакций.  
3. Управление списком каналов.  
4. Экспорт логов реакций.

## 6) Данные: как и что хранится

### 6.1 «Карточка ученика» как центральная сущность
1. Таблица `students` хранит профиль ученика и связь с Telegram owner.

### 6.3 Коммуникации и процессы
1. `student_requests` — все пользовательские заявки на модерацию.  
2. `events` — события/созвоны с метаданными и напоминаниями.  
3. `reminders`, `stage_reminder_rules`, `student_stage_reminders` — персональные и массовые напоминания.  
4. `alerts` — очередь уведомлений с повторами и статусами доставки/прочтения.

### 6.4 Контент и прогресс
1. `material_steps`, `material_topic_map` — структура roadmap и sync c Telegram group topics.  
2. `user_material_progress` — прогресс по шагам каждого пользователя.

### 6.5 Аналитика
1. `user_profiles` — роль и активность пользователя.  
2. `event_log` — события интерфейса и продуктовых воронок.

### 6.6 Рассылки
1. `broadcast_campaigns`, `broadcast_deliveries`, `auto_broadcast_rules` + read acknowledgements.

### 6.7 Owner `/t`
1. `owner_hh_jobs`, `owner_tg_accounts`, `owner_outreach_leads`, `owner_outreach_contacted`, `owner_outreach_parsed_pool`, `owner_outreach_events`, `owner_channel_reactions`.

### 6.8 Дополнительные сущности

## 7) Что запущено/предусмотрено на сервере (бот + прилегающие сервисы)

1. Telegram-бот как systemd unit `mentor-bot.service`.  
   Детали: auto-restart, env-файл, отдельный пользователь сервиса, защищенные параметры unit.

2. База данных SQLite в persistent storage (`/opt/mentor-bot/data/mentor_bot.db` по env-примеру).  

3. Internal API автооткликов (`hh-autoresponder`) для `POST /api/internal/accounts` и run endpoint.  

4. Interview Helper API-сервис (по умолчанию в env указан `http://85.198.82.221:8080`).  

5. Парсер HH (`AutoHh/parser_hh`) как часть owner-outreach контура.  

6. Опциональный Cloudflare OpenAI proxy для LLM-слоя `/t`.  

7. Статический сайт LeonovCare, который передает пользователя в бот с точным контекстом тарифа/маршрута.

## 8) Польза по ролям (бизнес-ценность)

### 8.1 Польза для обычного пользователя
1. Быстрый вход в нужный сценарий без ручного поиска по меню.  
2. Прозрачный путь: выбор формата -> договор -> оплата -> запуск сервиса.  
3. Диагностический созвон и Interview Helper доступны из одного интерфейса.

### 8.2 Польза для ученика
1. Единый личный кабинет с обучением, материалами, созвонами, моками, автооткликами и платежами.  
2. Понятный прогресс по roadmap и дисциплина действий через напоминания/слоты/ограничения.  
3. Меньше хаоса: все согласования и подтверждения проходят через единый модерационный поток.

### 8.3 Польза для ментора
1. Полноценная CRM-картина по ученику: карточка, платежи, созвоны, заявки, история взаимодействия.  
2. Сильная автоматизация рутины: scheduler, алерты, авторассылки, аналитика, экспорты.  
3. Owner `/t` масштабирует лидогенерацию и первичный outreach с AI-контролем и антидублями.
