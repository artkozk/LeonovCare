# Приведение предпросмотра к стандартной карточке (15.04.2026)

## Что изменено

1. В `app/bot.py` добавлен рендер предпросмотра `_student_card_preview_text(...)` в формате стандартной карточки:
   - заголовок `Ученик 1/1`;
   - ФИО, направление, этап, username, источник лида;
   - тариф, предоплата, правило постоплаты, статус постоплаты;
   - дата обучения, собесы, договор, автосообщения.
2. Вывод после успешной валидации договора переключен с технического списка `Что удалось заполнить` на стандартную карточку.
3. Технический формат `_prefill_text(...)` оставлен для внутреннего уведомления ментора.

## Поведение после изменения

1. Пользователь отправляет валидную ссылку на договор.
2. Бот пишет: `Ссылка валидна, договор распознан как наш.`
3. Следующим сообщением отправляет карточку в стандартном виде (`Ученик 1/1 ...`).
4. Если есть незаполненные поля, бот отдельно пишет `Нужно дозаполнить поля: ...` и дозапрашивает их по очереди.
