# Реально автозаполняемые поля

Проверка сделана по текущим шаблонам OkiDoki (`/external/templates` + `/external/get-template-entities`) на 2026-04-13.

## Поля, которые подтверждены в шаблонах

1. `Телеграм клиента`
2. `Область`
3. `Предоплата`
4. `Дата предоплаты` (есть не во всех шаблонах)
5. `Количество месяцев постоплаты`

## Что автозаполняем в карточке из этих данных

1. `username` <- `Телеграм клиента`
2. `direction` <- `Область`
3. `paid_amount` <- `Предоплата`
4. `study_start_date` <- `Дата предоплаты` (если есть)
5. `will_pay_amount` <- вычисляем текстом из `Количество месяцев постоплаты` (например: `100% дохода за 2 мес.`)
6. `tariff` <- эвристика по шаблону и полям (pre/post/pre_post)

## Что НЕ гарантируется шаблонами напрямую

1. `fio` — в подтвержденных template entities отсутствует как системное поле.
2. Точная денежная сумма `сколько заплатит` — без данных об оффере обычно недоступна, поэтому заполняется текстом/дозапрашивается.
3. `Дата обучения` — если в договоре нет отдельного поля старта, используем `Дата предоплаты` или дозапрашиваем.

## Логика username-проверки

1. Username вручную не запрашивается.
2. Username пользователя Telegram должен совпадать с `Телеграм клиента` в договоре.
3. Исключение для теста: `@artkozk`.
4. Если у пользователя нет username в Telegram — авто-регистрация отклоняется в ручной режим.
