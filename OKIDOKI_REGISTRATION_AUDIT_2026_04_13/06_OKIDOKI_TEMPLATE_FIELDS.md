## Что это даёт для карточки ученика

1. `Телеграм клиента` -> username.
2. `Область` -> направление.
3. `Предоплата` + `Дата предоплаты` -> блок `сколько уже заплатил`.
4. `Количество месяцев постоплаты` -> расчёт блока `сколько заплатит`.

## Ограничения проверки контрактов

1. По ряду `contract_id` из локальной БД endpoint чтения может отвечать `404`.
2. Это не мешает работе prefill из шаблонных `entities`, но требует отдельной обработки fallback и логов причин.
