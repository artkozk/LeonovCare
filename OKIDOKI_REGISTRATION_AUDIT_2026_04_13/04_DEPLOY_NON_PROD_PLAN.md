# 4. Выкатка в отдельный контур (без касания прода)

## Базовый принцип

1. Отдельный путь на сервере.
2. Отдельный процесс/сервис.
3. Отдельный токен бота и отдельная БД.
4. Продовый сервис не перезапускается.

## Минимальный план

1. Создать каталог:
- `/opt/mentor-bot-preprod`

2. Разместить туда только тестовый контур.

3. Поднять отдельный env:
- `BOT_TOKEN=<test_bot_token>`;
- `DB_PATH=mentor_bot_preprod.db`;
- `OKIDOKI_PROFILE_PREFILL_ENABLED=true`;
- `OKIDOKI_READ_ONLY_PROFILE_FLOW=true`.

4. Поднять отдельный systemd юнит:
- `mentor-bot-preprod.service`.

5. Логи писать отдельно:
- `/var/log/mentor-bot-preprod/`.

## Smoke тесты для preprod

1. `Я ученик -> валидная ссылка -> prefill`.
2. `Я ученик -> невалидная ссылка -> ручная регистрация`.
3. `Я ученик -> валидная, но не наш договор -> ручная регистрация + модерация`.
4. Проверка, что write-операции в OkiDoki не вызываются из этого потока.
